# CodeMerge Release Notes

本次发布聚焦于结果区预览稳定性修复、一次性合并规则支持，以及既有工作区 UI / 归档能力的持续整理。

## 更新内容

- 进一步修复点击“合并后内容”后应用无响应的问题：移除内容页入口和失效键计算中的大对象克隆/全量哈希，避免大结果下切换内容页时卡死。
- 左侧输入面板新增“本次合并规则”，支持手动添加目录 / 扩展名规则或导入 `.gitignore`，规则会参与预检和本次处理，成功后自动清空且不会写入配置。
- 新增大结果元数据场景下的内容页切换回归测试，覆盖真实用户路径下的响应性问题。
- 修复“合并后内容”预览在大结果下可能直接无响应的问题，恢复为虚拟列表渲染，避免一次性把整份合并文本灌入编辑器控件。
- 修复预览表与目录树在程序性同步选中时的重入链路，避免打开内容页或同步树选中时出现卡死。
- 重构主界面信息架构，拆分为输入区、状态区、结果区和规则管理区。
- 新增 Windows / macOS 自定义窗口标题栏，统一品牌区与状态胶囊；Linux 不支持 client-side decorations 时会自动退化为紧凑页内 header。
- 标题栏新增版本号显示与 GitHub 开源仓库入口，便于核对版本和访问项目主页。
- 新增窄窗口退化布局，避免三栏在小尺寸下挤压不可读。
- 将黑名单管理移出首页主流程，支持单条删除与更安全的危险操作确认。
- 优化处理中状态展示，增加摘要卡片、明确状态层和最近活动列表。
- 统一结果区文案与表头本地化，移除预览区面向最终用户无意义的调试信息。
- GitHub Release workflow 改为读取仓库内的 `RELEASE_NOTES.md` 作为发布说明。
- 修复 Linux `.deb` 安装后缺少桌面启动器的问题，现会一并安装 `.desktop` 文件和应用图标。
- 新增 macOS CI / Release workflow，按 `amd64` 与 `arm64` 产出 `CodeMerge.app` 的 `.zip` 和 `.dmg`。
- 新增 `.zip` 压缩包中的文本文件展开支持，可参与目录树、预检、预览和单文件合并。

## 构建产物

- Linux amd64: `.deb` 与 `.tar.gz`
- Windows amd64: `.zip` 与安装包
- macOS amd64 / arm64: `.zip`（`CodeMerge.app`）与 `.dmg`
- 修复处理结果临时目录清理不完整的问题，结果切换、重置和退出时现在会回收整次处理的临时产物目录。
- 修复超大“合并后内容”预览的并发点击竞态，加载 1MB / 加载全部现在会串行化，请求失败时会保留明确错误提示。
- 调整文件处理并发上限并移除归档元数据的无效计算，降低大批量处理时的额外 CPU 开销。
- 为配置加载/保存失败增加持久告警和重置/重试入口，并将合并输出中的目录结构、文件路径、字符数、Token 标签纳入 i18n。
- 预览表缓存改为基于结果修订号失效，快速滚动与文件列表拖拽也增加了节流，减少跳变和高频刷新。
- macOS 新增 `CODEMERGE_SYSTEM_TITLEBAR=1` 降级开关，可强制使用系统标题栏以规避不稳定环境。
